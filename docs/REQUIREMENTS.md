# 🚀 OpenWrt 性能监控可视化平台 — 需求规格说明书 v2.0

> **运行环境**：Unraid 虚拟机 · OpenWrt · 无无线功能
> **部署方式**：局域网内 Docker / 裸机部署

---

## 1. 项目定位

部署在局域网内，通过 SSH 采集 OpenWrt 路由器性能数据 + 局域网设备信息，提供实时、多维度的可视化监控平台。

**不做的事**：❌ 不改配置 ❌ 不管理固件 ❌ 不替代 LuCI ❌ 不管理无线

---

## 2. 功能需求

### 2.1 📊 仪表盘（Dashboard）

| 模块 | 内容 | 状态 |
|------|------|:----:|
| 实时指标卡片 | CPU / 内存 / 温度 / 上行流量 / 下行流量 | ✅ |
| 实时流量曲线 | WAN 口上下行速率折线图（最近 1 分钟） | ✅ |
| 系统负载 | load1 / load5 / load15 | ✅ |
| 连接跟踪 | conntrack 总数 / 最大数 / 使用率 | ✅ |
| 磁盘使用 | 各挂载点用量 + 百分比进度条 | ✅ |
| 单位切换 | b/s ↔ B/s 手动切换 | ✅ |
| 采集间隔 | 1s / 3s / 5s / 10s 可选 | ✅ |

### 2.2 🖥️ 设备管理

| 模块 | 内容 | 状态 |
|------|------|:----:|
| 添加设备 | IP / 端口 / 用户名 / 密码或密钥 | ✅ |
| 删除设备 | — | ✅ |
| 在线状态 | 绿点/红点 实时显示 | ✅ |
| 运行时间 | 设备 uptime 展示 | ✅ |

### 2.3 🌐 网络分析

| 模块 | 内容 | 状态 |
|------|------|:----:|
| 接口列表 | 所有网络接口（eth / br-lan / pppoe-wan 等） | ✅ |
| 接口详情 | 上下行总量、丢包、错误包 | ✅ |
| 连接协议分布 | IPv4 / IPv6 饼图 | ✅ |
| TCP 状态分布 | ESTAB / TIME_WAIT 等（需安装 ss） | ⏳ |

### 2.4 🖥️ 连接设备（新增！）

| 模块 | 内容 | 状态 |
|------|------|:----:|
| DHCP 租约列表 | 从 OpenWrt 读取所有 DHCP 分配的设备 | ❌ |
| ARP 表 | 读取 ARP 表补充未 DHCP 的设备 | ❌ |
| 设备信息 | IP 地址 / MAC 地址 / 主机名 / 租约剩余时间 | ❌ |
| 在线状态 | 根据 ARP 存活判断设备是否在线 | ❌ |
| 设备标记 | 手动备注设备名称（如"客厅电视"） | ❌ |

### 2.5 📈 CPU / 内存 深度分析（规划中）

| 模块 | 内容 | 状态 |
|------|------|:----:|
| CPU 历史曲线 | 每核使用率堆叠图，时间范围选择 | ❌ |
| 内存趋势 | 内存 / 缓存 / swap 面积图 | ❌ |
| 进程 TOP | CPU / 内存 占用最高的进程 | ❌ |

### 2.6 ⏱ 历史回溯（规划中）

| 模块 | 内容 | 状态 |
|------|------|:----:|
| 时序存储 | InfluxDB 存储历史数据 | ❌ |
| 时间选择器 | 回看 1h / 6h / 24h / 7d | ❌ |
| 对比模式 | 今天 vs 昨天 | ❌ |

### 2.7 🔔 智能告警（规划中）

| 规则 | 阈值 | 状态 |
|------|------|:----:|
| CPU 过高 | > 80% 持续 5min | ❌ |
| 内存不足 | 可用 < 15% | ❌ |
| 连接数异常 | 较前 5 分钟激增 > 3 倍 | ❌ |
| 接口断线 | 接口 down > 30s | ❌ |

## 3. 数据采集

### 3.1 系统层面（已实现）
- CPU 使用率（总/每核）
- 负载均衡（load1/5/15）
- 内存使用（总量/已用/缓存/可用）
- 磁盘使用（各挂载点）
- 运行时间
- 温度（无传感器则跳过）

### 3.2 网络层面（已实现）
- 各接口实时流量累计（RX/TX bytes）
- 连接跟踪数（conntrack_count）
- 连接跟踪协议分布（IPv4/IPv6）
- TCP 连接状态

### 3.3 局域网设备（待实现）
- DHCP 租约列表（`cat /tmp/dhcp.leases`）
- ARP 表（`cat /proc/net/arp`）
- 活跃连接（`conntrack -L` 按 IP 聚合）

## 4. 技术方案

```
[浏览器] ← WebSocket / REST → [FastAPI]
                                 ↓ SSH
                           [OpenWrt]
                           (Unraid VM)
```

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + Vite + ECharts + TailwindCSS |
| 后端 | Python FastAPI + asyncssh + SQLAlchemy |
| 通信 | WebSocket 实时推送 |
| 存储 | SQLite（当前）/ InfluxDB（规划） |
| 部署 | Docker Compose / 裸机 |
