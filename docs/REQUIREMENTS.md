# OpenWrt 性能监控可视化平台 — 需求规格说明书 v2.1

> 运行环境：局域网内 OpenWrt / Unraid / Docker / 裸机部署
> 当前版本：v0.4.2
> 最后更新：2026-06-25

## 1. 项目定位

部署在局域网内，通过 SSH 采集 OpenWrt 路由器性能、网络和局域网终端信息，提供实时、多维度、可长期运行的家庭网络监控平台。

边界：
- 不修改 OpenWrt 网络配置
- 不替代 LuCI
- 不管理固件升级
- HTTPS 由反向代理、Unraid 或 Nginx 处理

## 3. 数据采集需求

### 3.1 系统层面

- CPU 使用率，总体和每核
- load1 / load5 / load15
- 内存总量、已用、缓存、可用、swap
- 磁盘挂载点容量和使用率
- 温度传感器，设备无传感器时跳过
- 运行时间、固件版本、内核、架构、型号
- 进程 TOP

### 3.2 网络层面

- `/proc/net/dev` 接口流量、错误包、丢包
- conntrack 当前值和最大值
- conntrack 协议分布
- `/proc/net/tcp` TCP 状态分布
- OpenWrt WAN/LAN 接口地址
- nlbwmon 流量排行
- conntrack / iptables / nftables / iftop 兼容兜底

### 3.3 局域网终端

- DHCP 租约：`/tmp/dhcp.leases`
- ARP 表：`/proc/net/arp`
- 主机名、IP、MAC、接口、在线状态
- OUI 厂商识别
- 用户画像覆盖
- 自定义识别规则

## 5. 技术方案

```text
[浏览器 Vue 3]
      |
      | REST / WebSocket
      v
[FastAPI + SQLAlchemy + SQLite]
      |
      | asyncssh
      v
[OpenWrt]
```

| 层 | 技术 |
|----|------|
| 前端 | Vue 3 + Vite + ECharts + TailwindCSS |
| 后端 | FastAPI + asyncssh + SQLAlchemy |
| 实时通信 | WebSocket |
| 存储 | SQLite |
| 图标资产 | SVG / PNG / JPG / WebP 自定义图标 |
| 部署 | Docker Compose / GHCR 镜像 / Unraid 模板 / 裸机 |
